# Lo-Fi Beat Generator

An AI lo-fi beat generator using an LSTM sequence model trained on MIDI files.  
The backend generates melody notes, adds a lo-fi drum pattern, exports a MIDI file, converts it to WAV, and the frontend plays it directly in the browser.

---

## Stack

| Layer     | Tech                                              |
|-----------|---------------------------------------------------|
| ML model  | LSTM (TensorFlow/Keras)                           |
| Backend   | Python 3.11, Flask, PrettyMIDI, NumPy, Pandas     |
| Audio     | FluidSynth + midi2audio                           |
| Frontend  | Vanilla HTML / CSS / JS                           |

---

## Setup

### 1. Python environment

```bash
cd backend
python -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

### 2. FluidSynth & Soundfont (required for WAV conversion)

**macOS**
```bash
brew install fluidsynth
# Soundfont is bundled; or download FluidR3_GM.sf2 and set SOUNDFONT_PATH in config.py
```

**Ubuntu / Debian**
```bash
sudo apt install fluidsynth fluid-soundfont-gm
```

**Windows**  
Download FluidSynth from https://github.com/FluidSynth/fluidsynth/releases  
Download a soundfont (e.g. FluidR3_GM.sf2) and update `SOUNDFONT_PATH` in `backend/config.py`.

## Usage

### Step 1 вЂ” Add MIDI files

Place `.mid` or `.midi` files into:

```
backend/data/raw_midi/
```

Free MIDI sources: [midi.com](https://www.midi.com), [bitmidi.com](https://bitmidi.com), [MuseScore](https://musescore.com).  
Lo-fi/jazz/chill piano MIDI files work best.

### Step 2 вЂ” Preprocess

```bash
python -m backend.services.preprocess
```

Outputs: `backend/data/processed/notes.csv`

### Step 3 вЂ” Train

```bash
python -m backend.services.trainer
```

Outputs: `backend/models/lofi_model.keras`  
Training typically takes a few minutes on CPU with a small MIDI dataset.

### Step 4 вЂ” Run the backend

```bash
python backend/app.py
```

Server runs at `http://127.0.0.1:5000`

### Step 5 вЂ” Open the frontend

Open `frontend/index.html` directly in your browser, or serve it:

```bash
cd frontend
python -m http.server 8080
# Visit http://localhost:8080
```

Click **Generate Beat** вЂ” the backend generates a WAV and the player appears automatically.

## Configuration

All tunable values live in `backend/config.py`:

| Variable          | Default | Description                        |
|-------------------|---------|------------------------------------|
| `SEQUENCE_LENGTH` | 50      | Input window length for LSTM       |
| `EPOCHS`          | 50      | Max training epochs                |
| `BATCH_SIZE`      | 64      | Training batch size                |
| `LEARNING_RATE`   | 0.001   | Adam learning rate                 |
| `GENERATED_NOTES` | 300     | Notes generated per request        |
| `TEMPERATURE`     | 1.0     | Sampling randomness (higher = more varied) |
| `TEMPO`           | 90.0    | BPM for generated MIDI             |
| `SOUNDFONT_PATH`  | (auto)  | Path to .sf2 soundfont file        |

## Troubleshooting

- **"No .mid files found"** вЂ” Add MIDI files to `backend/data/raw_midi/`
- **"Model not found"** вЂ” Run `python -m backend.services.trainer` first
- **"WAV conversion failed"** вЂ” Install FluidSynth and verify the soundfont path in `config.py`
- **"Cannot reach backend"** вЂ” Make sure `python backend/app.py` is running
